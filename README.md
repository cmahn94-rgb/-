## 2. 작동 원리

### 전체 흐름

```
[1단계] 데이터 수집
        yfinance + 업비트 API로
        103개 종목 주가 1년치를 한꺼번에 다운로드
        ※ yfinance 세션 워밍업(crumb 사전 획득)으로 401 오류 방지

        ↓

[2단계] 시장 국면 감지 (market_phase.py)
        KOSPI의 MA20·MA60·RSI·VIX를 보고
        지금이 상승장인지 하락장인지 판단
        → 국면에 따라 최소 점수(2~6점) 자동 설정
        ※ 선행지표 포함: 10년물 금리(^TNX), 풋/콜 비율(^PCCE)

        ↓

[3단계] 신호 계산 (signals.py)
        103개 종목을 병렬로 분석
        ① RSI < 40 미충족이면 즉시 차단 (나머지 지표 계산 안 함)
        ② 기술지표 5개 + 보너스 5개로 점수 계산

        ↓

[4단계] 백테스트 검증 (signals.py)
        매수 신호 종목만 1년 과거 데이터로 검증
        아래 3가지 중 하나라도 해당되면 신호 차단:
        - 1년 백테스트 Sharpe < 0 (손실 전략)
        - Walk-forward 신뢰도 "낮음" (검증 구간 손실 또는 과적합)

        ↓

[5단계] 뉴스 수집 (data_loader.py)
        Gemini 실시간 검색 → Alpha Vantage → GNews → NewsAPI → yfinance
        5중 폴백 구조 / 영어 뉴스 자동 한국어 번역

        ↓

[6단계] AI 분석 (ai_analyst.py)
        Gemini가 시장 주요 이슈 3개 + 각 종목 신호 이유를 한 줄로 설명
        Gemini 429 오류 시 Groq(무료)로 자동 폴백

        ↓

[7단계] 리포트 조립 (scheduler_job.py)
        우선순위 점수(0~100점)로 종목 정렬 → TOP 3 선정
        진입 추천가 표시 (볼린저 하단 vs 현재가-ATR 중 높은 값)
        갭업 5% 이상이면 자동 제외

        ↓

[8단계] 텔레그램 전송 (telegram_bot.py)
        4000자씩 나눠서 전송
        신호 종목을 portfolio_signals.txt에 자동 기록
```

### 매수 신호 계산 방법

#### 전제 조건 (이것부터 통과해야 함)

| 조건 | 기준 | 이유 |
|------|------|------|
| **RSI < 40** | 미충족 시 즉시 차단 | RSI 40 이상은 과매도가 아닌 단순 하락 중. 과열 구간에서 매수 신호가 나는 근본 원인 차단 |

> RSI 40 이상인 종목은 나머지 5개 조건이 모두 충족돼도 신호가 뜨지 않습니다.

#### 기본 조건 (각 1점, 최대 6점)

| 번호 | 조건 | 쉬운 설명 |
|------|------|---------|
| c1 | RSI < 40 | 최근 2주 내린 날이 훨씬 많다 = 과매도 구간 |
| c2 | 20일 이동평균 상승 중 | 한 달 평균 가격이 오르는 추세 |
| c3 | 오늘 거래량 > 평균 × 1.3 | 평소보다 30% 이상 많이 거래 = 관심 급증 |
| c4 | 현재가가 볼린저 중심선 아래 | 통계적으로 평균보다 싼 가격 |
| c5 | MACD 히스토그램 음→양 전환 | 하락 에너지가 상승 에너지로 전환 |
| c6 | 변동성 돌파 + ADX ≥ 30 | 강한 추세와 함께 가격이 기준선 돌파 |

#### 보너스 조건 (각 +1점)

| 번호 | 조건 | 쉬운 설명 |
|------|------|---------|
| b1 | RSI 다이버전스 | 가격은 내리는데 RSI는 올라 = 반등 예고 |
| b2 | MACD + 거래량 복합 강세 | 두 강한 신호가 동시에 발생 |
| b3 | 52주 신고가 95% 이상 | 1년 최고점 근처 = 저항선 없는 구간 |
| b4 | MACD + 스토캐스틱 동시 골든크로스 | 두 지표 동시 전환 = 허위 신호 감소 |
| b5 | CCI < -100 | 통계적으로 극단적 저평가 (RSI와 이중 확인) |

#### 패널티

| 조건 | 효과 |
|------|------|
| 주간봉 MACD 음수 | 총점 -1점 (큰 흐름이 하락 추세) |
| 갭업 5% 이상 | 매수 목록에서 완전 제외 |
| 갭업 2~5% | 우선순위 점수 패널티 |
| 실적 발표 D-5 이내 | 우선순위 -5점 + 경고 표시 |

#### 백테스트 필터 (신호 차단 조건)

| 조건 | 차단 이유 |
|------|---------|
| 1년 Sharpe < 0 | 리스크 대비 수익이 마이너스인 전략 |
| WF 신뢰도 낮음 | 검증 구간 Sharpe < 0 또는 과적합 (Sharpe 차이 > 0.5) |

#### 시장 국면별 최소 점수

실행할 때마다 자동으로 결정됩니다.

| 국면 | 최소 점수 | 상황 |
|------|---------|------|
| 🚀 강한 상승 | **2점** | KOSPI가 강하게 오르는 중 |
| 📈 완만한 상승 | **3점** | 보통 상승 추세 |
| ↔️ 횡보 박스권 | **4점** | 오르지도 내리지도 않음 |
| 📉 조정 하락 | **5점** | 조금씩 내리는 중 |
| 🔴 급락 패닉 | **6점** | 급격히 하락 (거의 신호 없음) |

#### 시장 국면 선행지표

후행 지표(MA, VIX)만 쓰면 하락장이 이미 시작된 다음에 경고가 옵니다.  
아래 선행지표 2개를 추가해 1~4주 일찍 경고합니다.

| 지표 | 경보 조건 | 의미 |
|------|---------|------|
| 10년물 국채금리(^TNX) | 한 달 새 +0.3%p 이상 급등 | 기업 대출비용 상승 → 주가 압박 선행 |
| 풋/콜 비율(^PCCE) | 0.8 이상 또는 5일 평균 대비 +20% | 투자자 다수가 하락 헤지 = 선행 공포 신호 |

#### 우선순위 종합점수 (0~100점)

신호가 여러 개일 때 어떤 종목을 먼저 보여줄지 정하는 점수입니다.

| 항목 | 최대 점수 | 설명 |
|------|---------|------|
| 신호 점수 | 25점 | 기술지표 점수를 0~25로 환산 |
| RSI 점수 | 20점 | RSI 낮을수록 (20 이하 = 20점, 40 미만 = 10점 내외) |
| Sharpe 점수 | 25점 | 백테스트 위험 대비 수익률 |
| 승률 점수 | 20점 | 백테스트 승률 |
| ADX 점수 | 10점 | 추세 강도 |

#### 진입 추천가

신호 발생 = 지금 당장 시가 매수가 아닙니다.  
알림에 지정가 대기 기준을 함께 표시합니다.

```
• 📌 진입 추천가: ₩285,000 (현재가 대비 -2.3%, 지정가 대기)
     └ 볼린저하단 ₩283,000 | 현재가-ATR ₩285,000
```

- **볼린저 하단**: 통계적 저점 지지선
- **현재가 - ATR**: 하루 변동폭만큼 눌렸을 때
- 두 값 중 현재가에 더 가까운 값을 추천 진입가로 제시합니다.

### 설치 순서

#### 1단계: 코드를 GitHub에 올리기

```bash
git init
git add .
git commit -m "퀀트봇 초기 설정"
git remote add origin https://github.com/내아이디/내레포이름.git
git push origin main
```

#### 2단계: GitHub Secrets 등록

GitHub 레포지토리 페이지 → **Settings → Secrets and variables → Actions → New repository secret**

| 이름 (Name) | 값 (Secret) |
|------------|------------|
| `TELEGRAM_TOKEN` | 텔레그램 봇 토큰 |
| `TELEGRAM_CHAT_ID` | 내 텔레그램 채팅 ID |
| `GEMINI_API_KEY` | Gemini API 키 |
| `GROQ_API_KEY` | Groq API 키 (**필수 권장** — Gemini 429 자동 폴백) |
| `ALPHAVANTAGE_API_KEY` | Alpha Vantage 키 (선택 — 미국주식 뉴스) |
| `GNEWS_API_KEY` | GNews API 키 (선택 — 한국주식 뉴스) |
| `NEWSAPI_KEY` | NewsAPI 키 (선택 — 뉴스 폴백) |
| `DART_API_KEY` | DART API 키 (선택 — 한국주식 기본적 분석) |

> **텔레그램 채팅 ID 확인하는 법**  
> 1. 만든 봇에 아무 메시지나 보내기  
> 2. 브라우저에서 아래 주소 접속 (토큰 부분을 본인 것으로 교체)  
>    `https://api.telegram.org/bot여기에토큰입력/getUpdates`  
> 3. `"chat":{"id": 숫자}` 에서 숫자 부분이 채팅 ID

#### 3단계: 자동 실행 확인

GitHub 레포지토리 → **Actions 탭**에서 워크플로우가 활성화돼 있는지 확인합니다.

이제 매일 KST **09:00 / 15:30 / 23:30**에 자동으로 실행됩니다.

> ⚠️ **cron-job.org 등 외부 트리거를 함께 사용하면 새벽에 알림이 옵니다.**  
> 이 파일의 GitHub Actions 내장 cron만 사용하고, 외부 트리거는 모두 삭제하세요.

수동 테스트: Actions → Run analysis → Run workflow 클릭

## 5. HTML 대시보드 설정

텔레그램 메시지는 텍스트 서식만 지원해서 한눈에 보기 불편합니다. HTML 대시보드를 설정하면 실행할 때마다 **보기 좋은 웹 리포트**가 GitHub Pages에 자동 배포되고, 텔레그램에는 링크가 함께 전송됩니다.

```
텔레그램 메시지 하단에 추가되는 내용:
─────────────────────────
📊 HTML 대시보드
  2026-06-02-0902.html  ← 클릭하면 오늘 리포트 (예쁜 화면)
  전체 목록 보기        ← 클릭하면 날짜별 히스토리 목록
```

### 설정 방법 (최초 1회)

#### 1단계: GitHub Pages 활성화

레포지토리 → **Settings → Pages**  
→ Source: `Deploy from a branch`  
→ Branch: `main` / `docs`  
→ **Save**

잠시 후 `https://[계정명].github.io/[레포명]` 주소가 활성화됩니다.

#### 2단계: GH_PAGES_URL Secret 등록

Settings → Secrets and variables → Actions → **New repository secret**

| 이름 | 값 |
|------|---|
| `GH_PAGES_URL` | `https://[계정명].github.io/[레포명]` |

이 Secret을 등록하면 텔레그램 메시지에 HTML 링크가 자동으로 붙습니다.  
등록하지 않으면 HTML 파일은 생성되지만 링크는 전송되지 않습니다.

> **KRX·네이버금융 수급 API처럼 별도 API 키는 필요 없습니다.** GitHub 계정만 있으면 됩니다.

### 히스토리 관리

실행할 때마다 `docs/YYYY-MM-DD-HHMM.html` 파일이 쌓입니다.  
인덱스 페이지(`index.html`)에는 최근 90개(약 1달치)가 표시됩니다.

**특정 날짜 삭제**

```bash
git rm docs/2026-06-01-0900.html
git commit -m "리포트 삭제"
git push
```

**전체 초기화**

```bash
git rm docs/*.html
git commit -m "히스토리 초기화"
git push
```

> 초기화 후 다음 실행 때 `index.html`이 자동 재생성됩니다.  
> index.html 하단에도 삭제 명령어가 항상 표시되어 있습니다.

### 용량 안내

HTML 파일 1개 ≈ 50~100KB. 하루 3번 × 평일 250일 = 연간 약 75MB.  
GitHub 무료 플랜 용량 한도(1GB)에서 13년치를 저장해도 여유 있습니다.

## 6. 설정 변경 방법

`settings.txt` 파일을 메모장으로 열어서 숫자만 바꾸면 됩니다.  
바꾼 뒤 GitHub에 push하면 다음 실행부터 적용됩니다.

```
RSI_BUY=40
```
RSI가 이 값 **미만**인 종목만 매수 신호를 냅니다.  
**미충족 시 신호 자체가 차단됩니다** (임계값 상향이 아닌 완전 차단).  
신호가 너무 없으면 45로 올려보세요. 50 이상은 권장하지 않습니다.

```
RSI_SELL=75
```
RSI가 이 값을 초과하면 매도 신호입니다.

```
STOP_LOSS=-5
```
진입가 대비 이 % 이상 내리면 손절 알림입니다.

```
TRAILING_STOP=8
```
매수 후 최고점 대비 이 % 내리면 청산 알림입니다.

```
TARGET_1=12
TARGET_2=25
```
분할 매도 목표입니다. 1차 12%, 2차 25% 수익 시 알림.

```
ADX_MIN=30
```
변동성 돌파 조건(c6)이 인정되는 최소 ADX 추세 강도입니다.  
낮출수록 신호가 늘지만 허위 신호도 증가합니다. 25~35 권장.

```
MAX_HOLDING_DAYS=30
```
이 일수 이상 보유 중이면 청산 권고 알림이 옵니다.

```
MAX_SIGNAL_DISPLAY=10
```
알림에 상세 표시할 최대 종목 수입니다.

```
BUY_SCORE_THRESHOLD=4
```
매수 최소 점수 기본값입니다. 실제 실행 시 시장 국면이 자동으로 덮어씁니다.

```
CORRELATION_MAX=0.7
```
매수 후보 종목 간 상관계수 상한입니다. 이 값 이상이면 우선순위 낮은 종목을 제외합니다. 반도체 동시 신호처럼 위장된 집중 투자를 방지합니다. 0.8로 올리면 더 허용적, 0.6으로 낮추면 더 엄격합니다.

```
MAX_POSITIONS=8
```
동시 매수 권장 최대 종목 수입니다 (주식 기준, 크립토 별도). 초과하면 우선순위 낮은 종목은 제외됩니다.

## 8. 포트폴리오 관리

### portfolio.txt — 내가 보유한 종목

실제로 산 종목을 여기에 적으면 매도 신호와 손익 알림을 받습니다.  
**이 파일에 없는 종목은 매도 신호가 표시되지 않습니다.**

```
형식: 티커,보유수량,평균단가,통화

예시:
005930.KS,10,68000,KRW
AAPL,5,180.50,USD
UPBIT:BTC,0.01,95000000,KRW
```

### portfolio_signals.txt — 신호 자동 기록

매수 신호가 날 때마다 자동으로 여기에 기록됩니다.

```
형식: 날짜,티커,종목명,신호점수,진입예상가,통화,보유일수,상태
```

**상태 종류:**

| 상태 | 의미 | 변경 방법 |
|------|------|---------|
| `신호발생` | 신호는 났지만 아직 사지 않음 | 자동 기록 |
| `보유중` | 실제로 산 경우 | **직접 수정** |
| `익절` | 목표가 도달 후 판 경우 | 직접 수정 |
| `손절` | 손절가 도달 후 판 경우 | 직접 수정 |
| `기간초과청산` | 30일 지나서 판 경우 | 직접 수정 |

**사용 방법:**
1. 신호가 오면 `신호발생`으로 자동 기록됩니다
2. 실제로 샀다면 해당 줄의 상태를 `보유중`으로 직접 바꿉니다
3. 30일이 지나면 자동으로 "청산 권고" 알림이 옵니다
4. 매주 일요일에 지난 주 신호들의 실제 수익률 요약이 옵니다

**주간 성과 리포트 예시 (매주 일요일 자동 발송):**
```
📊 주간 신호 성과 리포트
기간: 2026-05-05 ~ 2026-05-12
총 7개 신호 | 수익 4개(57%) | 손실 3개(43%)
평균 수익률: +3.2%
최고: +8.1% (AMD) | 최저: -2.4% (INTC)
```

## 9. 자주 묻는 질문

**Q. 매수 신호가 하나도 안 나와요.**

RSI 40 미만인 종목이 없는 상승장이면 정상입니다.  
시장 국면이 "강한 상승"으로 판단되면 임계값이 2점으로 낮아집니다.  
그래도 없으면 `settings.txt`에서 `RSI_BUY=45`로 올려보세요.  
50 이상으로 올리는 건 권장하지 않습니다.

---

**Q. Walk-forward에서 "신뢰도 낮음"으로 차단되는 종목이 많아요.**

정상입니다. RSI 40 기준이 엄격해지면서 거래 횟수가 줄고,  
검증 구간 거래가 부족하거나 Sharpe가 음수인 경우 차단됩니다.  
이는 실전에서도 손실 날 가능성이 높은 전략을 걸러내는 것입니다.

---

**Q. "Gemini 429 오류"가 자꾸 납니다.**

Gemini 무료 티어는 분당 15회 한도입니다.  
GitHub Secrets에 `GROQ_API_KEY`를 등록하면 자동으로 Groq(분당 30회, 무료)로 폴백됩니다.  
[console.groq.com](https://console.groq.com)에서 이메일만 입력하면 무료 발급됩니다.

---

**Q. 알림이 새벽 4시에 옵니다.**

cron-job.org 등 외부 트리거 서비스를 UTC/KST 혼동해서 설정하면  
예상과 9시간 다른 시각에 실행됩니다.  
cron-job.org에 등록된 트리거를 **모두 삭제**하고  
이 파일의 GitHub Actions 내장 cron만 사용하세요.

---

**Q. 알림에 뉴스가 안 보여요.**

뉴스는 5중 폴백으로 작동합니다.

1. Gemini Grounding (변동률 ±2% 이상 종목만)
2. Alpha Vantage (미국 주식 전용, `ALPHAVANTAGE_API_KEY` 필요)
3. GNews (한국·미국 전종목, `GNEWS_API_KEY` 필요 — [gnews.io](https://gnews.io) 무료)
4. NewsAPI (GNews 폴백, `NEWSAPI_KEY` 필요 — [newsapi.org](https://newsapi.org) 무료)
5. yfinance (최후 수단, 불안정)

한국 주식 뉴스는 GNews(`GNEWS_API_KEY`)가 없으면 yfinance에만 의존해 뉴스가 0개로 올 수 있습니다.

---

**Q. HTML 대시보드 링크가 텔레그램에 안 붙어요.**

`GH_PAGES_URL` Secret이 등록되지 않은 경우입니다. Settings → Secrets에서 `GH_PAGES_URL` 값을 `https://[계정명].github.io/[레포명]` 형식으로 등록하세요. 또한 GitHub Actions가 `docs/` 폴더에 쓰기 권한(`contents: write`)을 가져야 합니다 — `run_analysis.yml`에 이미 설정되어 있습니다.

---

**Q. 알림이 너무 많아요.**

`settings.txt`에서 두 가지를 조정하세요.

- `MAX_SIGNAL_DISPLAY=5` (상세 표시 줄이기)
- `BUY_SCORE_THRESHOLD=5` (기준 높이기)

---

**Q. 백테스트 수익률이 실제와 다를 것 같아요.**

맞습니다. 백테스트는 "신호 발생 당일 종가"에 산다고 가정하는데  
실제로는 다음 날 시가에 삽니다.  
갭이 1~3% 발생하고 슬리피지도 더 크게 나옵니다.  
**실제 수익률은 백테스트보다 20~40% 낮게** 나오는 게 일반적입니다.  
반드시 페이퍼 트레이딩으로 먼저 검증하세요.

---

**Q. 이 프로그램이 자동으로 주식을 사주나요?**

아닙니다. **알림만** 보내줍니다.  
실제 매매는 직접 증권사 앱에서 해야 합니다.

### v5.2 — 기관/외국인 수급 팩터

한국 주식시장은 기관·외국인 수급이 주가에 미치는 영향이 매우 큽니다. 개인이 살 때 기관·외국인이 팔면 단기 반등이 꺾이는 패턴이 반복됩니다.

**데이터 소스 (2중 폴백)**

```

        ↓ 실패 시
네이버금융 비공식 API — 폴백
```

**수급 점수 기준**

| 조건 | 점수 | 의미 |
|------|------|------|
| 외국인 3일 이상 연속 순매수 | +1 | 스마트머니 유입 |
| 기관 3일 이상 연속 순매수 | +1 | 대형 매수세 |
| 외국인+기관 당일 동시 순매수 | +1 | 가장 강력한 매수 근거 |
| 외국인+기관 동시 연속 매도 3일+ | -1 | 매수 신호 신뢰도 하락 |

> 수급 팩터는 **한국 주식(KR)에만** 적용됩니다. 미국 주식·크립토는 해당 없음.

**알림 예시**

```
• 💚 수급: 외국인 5일 연속매수 | 기관 4일 연속매수 | 외국인+기관 동시매수
• 🔻 수급: 외국인·기관 동반매도 4일 — 수급 패널티 -1점 적용
• ➡️ 수급: 데이터 조회 실패 (KRX/네이버 API) — 점수 미반영
```

### v5.1 — 리스크 관리 및 팩터 독립성

#### 🔴 포트폴리오 집중 위험 차단 (1순위)

| 항목 | 내용 |
|------|------|
| 상관관계 필터 | 매수 후보 간 최근 60일 일간수익률 상관계수가 `CORRELATION_MAX`(기본 0.7) 이상이면 우선순위 낮은 종목 제외. 반도체 4종목 동시 신호 같은 위장된 집중 투자 방지 |
| 최대 보유 종목 수 | `MAX_POSITIONS`(기본 8) 초과 시 추가 종목 제외 — 과분산 방지 |

> 같은 통화권(KR끼리, US끼리)만 비교하며, 크립토는 면제됩니다.

#### 🟡 백테스트 표본 신뢰성 (2순위)

| 항목 | 내용 |
|------|------|
| 거래 표본 하한 | 1년 백테스트 거래 횟수 5회 미만이면 Sharpe·승률 점수를 0 처리. 거래 3회짜리 Sharpe 2.0은 운이지 실력이 아니므로 우선순위에 반영하지 않음 |

#### 🟢 팩터 중복 제거 (4순위)

종가에서 파생된 RSI·MACD·MA·볼린저가 같은 정보를 중복 카운트하는 문제를 개선했습니다. 6개 조건을 4개 독립 축으로 재분류합니다.

| 독립 축 | 구성 조건 | 측정 대상 |
|--------|---------|---------|
| 추세축 | MA20 상승 OR 변동성 돌파 | 가격 방향 |
| 모멘텀축 | MACD 전환 | 추세 전환 속도 |
| 평균회귀축 | RSI 과매도 OR 볼린저 하단 | 저평가 |
| 수급축 | 거래량 급증 | 참여자 관심 |

- 3개 이상 독립 축 동시 충족 시 보너스 +1점
- 우선순위 신호점수에 축 다양성 가중 (같은 축 4번 충족한 4점 < 독립 축 3개 충족한 3점)
- **강력 매수** 판정에 "독립 축 3개 이상" 요건 추가

### v5.0 — 전략 신뢰도 개선

#### 🔴 치명적 결함 수정

| 항목 | 수정 전 | 수정 후 |
|------|-------|-------|
| RSI 진입 기준 | RSI < 50 (단순 하락 중인 종목도 통과) | **RSI < 40 미충족 시 즉시 차단** (과매도 기준 명확화) |
| Sharpe 음수 종목 | 경고 없이 신호 발생 | **Sharpe < 0이면 신호 차단** (손실 전략 걸러냄) |
| WF 거래 부족 | "차단 보류, 참고용"으로 통과 | **WF 신뢰도 낮음 = 완전 차단** (검증 불가 = 신뢰 불가) |
| 진입 타이밍 | 신호 당일 시가 매수 암묵적 가정 | **진입 추천가 표시** (볼린저 하단 / 현재가-ATR) |

#### 🟡 전략 강화

| 항목 | 내용 |
|------|------|
| 국면 선행지표 | 10년물 금리(^TNX) + 풋/콜 비율(^PCCE) 추가 — 하락장 1~4주 선행 경고 |
| WF 신뢰도 기준 강화 | 검증 Sharpe < 0이면 과적합 여부와 무관하게 "낮음"으로 처리 |
| RSI 우선순위 점수 | RSI 40 이하 구간으로 점수 테이블 재조정 |
| 시간대별 필터 | 09:00·15:30은 KR+크립토만, 23:30은 전체 분석 |
