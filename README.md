## 2. 작동 원리

### 전체 흐름

```
[1단계] 데이터 수집
        yfinance + 업비트 API로
        103개 종목 주가 1년치를 한꺼번에 다운로드

        ↓

[2단계] 시장 국면 감지 (market_phase.py)
        KOSPI의 MA20·MA60·RSI·VIX를 보고
        지금이 상승장인지 하락장인지 판단
        → 국면에 따라 최소 점수(2~6점) 자동 설정

        ↓

[3단계] 신호 계산 (signals.py)
        103개 종목을 20개씩 동시에 분석
        기술지표 6개 + 보너스 5개로 점수 계산
        RSI가 과열이면 기준을 1점 더 높게 요구

        ↓

[4단계] 백테스트 검증 (signals.py)
        매수 신호 종목만 1년 과거 데이터로 검증
        Walk-forward: 앞 8개월 학습 → 뒤 4개월 검증
        과적합 + 검증 Sharpe < 0.3 이면 신호 차단

        ↓

[5단계] 뉴스 수집 (data_loader.py)
        Gemini 실시간 검색 → Alpha Vantage → yfinance
        3중 폴백 구조 (하나 실패해도 다음 소스 사용)
        영어 뉴스는 자동으로 한국어 번역

        ↓

[6단계] AI 분석 (ai_analyst.py)
        Gemini가 각 종목 신호 이유를 한 줄로 설명
        Gemini 429 오류 시 Groq(무료)로 자동 폴백
        시장 주요 이슈 3개 실시간 검색

        ↓

[7단계] 리포트 조립 (scheduler_job.py)
        우선순위 점수(0~100점)로 종목 정렬
        TOP 3 선정, 순위 뱃지(🥇🥈🥉) 부여
        갭업 5% 이상이면 자동 제외

        ↓

[8단계] 텔레그램 전송 (telegram_bot.py)
        Markdown 특수문자 자동 이스케이프
        4000자씩 나눠서 전송
        신호 종목을 portfolio_signals.txt에 기록
```

### 매수 점수 계산 방법

#### 기본 조건 (각 1점, 최대 6점)

| 번호 | 조건 | 쉬운 설명 |
|------|------|---------|
| c1 | RSI < 50 | 최근 2주 중 내린 날이 더 많다 = 저평가 가능성 |
| c2 | 20일 이동평균 상승 중 | 최근 한 달 평균 가격이 오르는 추세 |
| c3 | 오늘 거래량 > 평균 × 1.3 | 평소보다 30% 이상 많이 거래됨 = 관심 급증 |
| c4 | 현재가가 볼린저 중심선 아래 | 통계적으로 평균보다 싼 가격 |
| c5 | MACD 히스토그램 음→양 전환 | 단기 추세가 장기 추세를 위로 돌파 |
| c6 | 변동성 돌파 + ADX ≥ 30 | 강한 추세와 함께 가격이 기준선 돌파 |

> **RSI 예외**: RSI 조건(c1)이 충족 안 되면 최소 점수가 1점 높아집니다.
> RSI 70 이상인 과열 종목이 다른 조건만으로 신호나는 걸 막기 위해서입니다.

#### 보너스 조건 (각 +1점)

| 번호 | 조건 | 쉬운 설명 |
|------|------|---------|
| b1 | RSI 다이버전스 | 가격은 내리는데 RSI는 올라 = 반등 예고 |
| b2 | MACD + 거래량 복합 강세 | 두 강한 신호가 동시에 발생 |
| b3 | 52주 신고가 95% 이상 | 1년 최고점 근처 = 저항선 없는 구간 |
| b4 | MACD + 스토캐스틱 동시 골든크로스 | 두 지표 동시 전환 = 허위 신호 감소 |
| b5 | CCI < -100 | 통계적으로 극단적 저평가 |

#### 패널티

| 조건 | 효과 |
|------|------|
| 주간봉 MACD 음수 | 총점 -1점 (큰 흐름이 하락 추세) |
| RSI 조건 미충족 | 최소 점수 +1점 요구 |
| 갭업 5% 이상 | 매수 목록에서 제외 |
| 갭업 2~5% | 우선순위 점수 패널티 |

#### 시장 국면별 최소 점수

실행할 때마다 자동으로 결정됩니다. `settings.txt`의 `BUY_SCORE_THRESHOLD`는 기본값이고, 아래 표로 덮어씌워집니다.

| 국면 | 최소 점수 | 상황 |
|------|---------|------|
| 🚀 강한 상승 | **2점** | KOSPI가 강하게 오르는 중 |
| 📈 완만한 상승 | **3점** | 보통 상승 추세 |
| ↔️ 횡보 박스권 | **4점** | 오르지도 내리지도 않음 |
| 📉 조정 하락 | **5점** | 조금씩 내리는 중 |
| 🔴 급락 패닉 | **6점** | 급격히 하락 (거의 신호 없음) |

#### 우선순위 종합점수 (0~100점)

신호가 여러 개일 때 어떤 종목을 먼저 보여줄지 정하는 점수입니다.

| 항목 | 최대 점수 | 설명 |
|------|---------|------|
| 신호 점수 | 25점 | 위 기술지표 점수를 0~25로 환산 |
| RSI 점수 | 20점 | RSI 낮을수록(저평가) 높은 점수 |
| Sharpe 점수 | 25점 | 백테스트 위험 대비 수익률 |
| 승률 점수 | 20점 | 백테스트 승률 |
| ADX 점수 | 10점 | 추세 강도 |

## 4. 설치 방법

### 필요한 것 (모두 무료)

| Groq API 키 | console.groq.com | 권장 (429 방지) |
| Alpha Vantage 키 | alphavantage.co | 선택 (뉴스 품질) |

### 설치 순서

#### 1단계: 코드를 GitHub에 올리기

이 폴더 전체를 GitHub 레포지토리에 올립니다.

```bash
git init
git add .
git commit -m "퀀트봇 초기 설정"
git remote add origin https://github.com/내아이디/내레포이름.git
git push origin main
```

#### 2단계: GitHub Secrets 등록

GitHub 레포지토리 페이지에서

**Settings → Secrets and variables → Actions → New repository secret**

아래 항목을 하나씩 등록합니다.

| 이름 (Name) | 값 (Secret) |
|------------|------------|
| `TELEGRAM_TOKEN` | 텔레그램 봇 토큰 |
| `TELEGRAM_CHAT_ID` | 내 텔레그램 채팅 ID |
| `GEMINI_API_KEY` | Gemini API 키 |
| `GROQ_API_KEY` | Groq API 키 (권장) |
| `ALPHAVANTAGE_API_KEY` | Alpha Vantage 키 (선택) |

> **텔레그램 채팅 ID 확인하는 법**
> 1. 만든 봇에 아무 메시지나 보내기
> 2. 브라우저에서 아래 주소 접속 (토큰 부분은 본인 것으로 교체)
>    `https://api.telegram.org/bot여기에토큰입력/getUpdates`
> 3. `"chat":{"id": 숫자}` 에서 숫자 부분이 채팅 ID

#### 3단계: 자동 실행 확인

GitHub 레포지토리 → **Actions 탭**에서
워크플로우가 활성화돼 있는지 확인합니다.

이제 매일 KST 09:00 / 15:30 / 23:30에 자동으로 실행됩니다.
수동으로 바로 테스트하려면 Actions → Run analysis → Run workflow 클릭.

## 5. 설정 변경 방법

`settings.txt` 파일을 메모장으로 열어서 숫자만 바꾸면 됩니다.
바꾼 뒤 GitHub에 push하면 다음 실행부터 적용됩니다.

```
RSI_BUY=50
```
RSI가 이 값보다 낮은 종목만 매수 신호를 냅니다.
낮출수록 더 엄격해져 신호가 줄고, 높일수록 신호가 늘어납니다.
급등장에서 신호가 0개라면 55로 올려보세요.

```
RSI_SELL=75
```
RSI가 이 값을 초과하면 매도 신호입니다.
낮출수록 더 빨리 팔라고 알려줍니다.

```
STOP_LOSS=-5
```
진입가 대비 이 % 이상 내리면 손절 알림입니다.
예: -5이면 5% 내리면 알림.

```
TRAILING_STOP=8
```
매수 후 최고점 대비 이 % 내리면 청산 알림입니다.
예: 8이면 고점에서 8% 빠지면 알림.

```
TARGET_1=12
TARGET_2=25
```
분할 매도 목표입니다.
1차로 12% 오르면, 2차로 25% 오르면 알림.

```
MAX_HOLDING_DAYS=30
```
이 일수 이상 보유 중이면 청산 권고 알림이 옵니다.
portfolio_signals.txt에서 상태를 "보유중"으로 바꿔야 카운트가 시작됩니다.

```
MAX_SIGNAL_DISPLAY=10
```
알림에 상세 표시할 최대 종목 수입니다.
초과하는 종목은 간략 요약으로 표시됩니다.

```
BUY_SCORE_THRESHOLD=4
```
매수 최소 점수입니다. 하지만 실제 실행 시 시장 국면이 자동으로 덮어씁니다.
(강한상승=2점, 횡보=4점, 패닉=6점)

## 6. 종목 추가/삭제

`stocks.txt` 파일을 메모장으로 엽니다.

### 형식

```
티커코드,종목명,시장구분
```

### 시장 구분 종류

| 코드 | 의미 |
|------|------|
| `KR` | 한국 주식 (`.KS` 또는 `.KQ` 붙은 티커) |
| `US` | 미국 주식 |
| `CRYPTO_KRW` | 업비트 원화 코인 |

### 예시

```
# 한국 주식 추가
373220.KS,LG에너지솔루션,KR

# 미국 주식 추가
TSLA,테슬라,US

# 업비트 코인 추가
UPBIT:ETH,이더리움,CRYPTO_KRW

# 이 줄 앞에 # 붙이면 감시에서 제외됨
# 005935.KS,삼성전자우,KR
```

> ⚠️ 종목을 너무 많이 추가하면 실행 시간이 길어질 수 있습니다.
> 현재 103개 기준 약 1~2분 소요됩니다.

## 7. 포트폴리오 관리

### portfolio.txt — 내가 보유한 종목

실제로 산 종목을 여기에 적으면 매도 신호와 손익 알림을 받습니다.
**이 파일에 없는 종목은 매도 신호가 표시되지 않습니다.**

```
형식: 티커,보유수량,평균단가,통화

예시:
005930.KS,10,68000,KRW
AAPL,5,180.50,USD
UPBIT:BTC,0.01,95000000,KRW
```

### portfolio_signals.txt — 신호 자동 기록

매수 신호가 날 때마다 자동으로 여기에 기록됩니다.

```
형식: 날짜,티커,종목명,신호점수,진입예상가,통화,보유일수,상태
```

**상태 종류:**

| 상태 | 의미 | 변경 방법 |
|------|------|---------|
| `신호발생` | 신호는 났지만 아직 사지 않음 | 자동 기록 |
| `보유중` | 실제로 산 경우 | **직접 수정** |
| `익절` | 목표가 도달 후 판 경우 | 직접 수정 |
| `손절` | 손절가 도달 후 판 경우 | 직접 수정 |
| `기간초과청산` | 30일 지나서 판 경우 | 직접 수정 |

**사용 방법:**
1. 신호가 오면 `신호발생`으로 자동 기록됩니다
2. 실제로 샀다면 해당 줄의 상태를 `보유중`으로 직접 바꿉니다
3. 30일이 지나면 자동으로 "청산 권고" 알림이 옵니다
4. 매주 일요일에 지난 주 신호들의 실제 수익률 요약이 옵니다

**주간 성과 리포트 예시 (매주 일요일 자동 발송):**
```
📊 주간 신호 성과 리포트
기간: 2026-05-05 ~ 2026-05-12
총 7개 신호 | 수익 4개(57%) | 손실 3개(43%)
평균 수익률: +3.2%
최고: +8.1% (AMD) | 최저: -2.4% (INTC)
```

## 8. 자주 묻는 질문

**Q. 매수 신호가 하나도 안 나와요.**

시장 전체가 RSI 50 이상인 급등장이면 정상입니다.
시장 국면이 "강한 상승"으로 판단되면 임계값이 2점으로 낮아져 신호가 늘어납니다.
그래도 없으면 `settings.txt`에서 `RSI_BUY=55`로 올려보세요.

---

**Q. "Gemini 429 오류"가 자꾸 납니다.**

Gemini 무료 티어는 분당 15회 한도입니다.
Groq API 키를 GitHub Secrets에 `GROQ_API_KEY`로 등록하면
Gemini 실패 시 자동으로 Groq(분당 30회, 무료)로 넘어갑니다.
[console.groq.com](https://console.groq.com)에서 이메일만 입력하면 무료 발급됩니다.

---

**Q. 알림에 뉴스가 안 보여요.**

뉴스는 3중 폴백으로 작동합니다.

1. Gemini Grounding (변동률 ±2% 이상 종목만)
2. Alpha Vantage (미국 주식만, `ALPHAVANTAGE_API_KEY` 필요)
3. yfinance (불안정, 최후 수단)

한국 주식 뉴스는 Alpha Vantage가 지원하지 않아 yfinance에 의존합니다.
Yahoo Finance API가 자주 바뀌어서 뉴스가 0개로 오는 경우가 있습니다.
이건 외부 서비스 문제라 코드 수정으로 완전히 해결하기 어렵습니다.

---

**Q. 알림이 너무 많아요.**

`settings.txt`에서 두 가지를 조정하세요.

- `MAX_SIGNAL_DISPLAY=5` (상세 표시 줄이기)
- `BUY_SCORE_THRESHOLD=5` (기준 높이기)

---

**Q. 백테스트 수익률이 실제와 다를 것 같아요.**

맞습니다. 백테스트는 "신호 발생 당일 종가"에 산다고 가정하는데
실제로는 다음 날 시가에 삽니다.
갭이 1~3% 발생하고 슬리피지도 더 크게 나옵니다.
**실제 수익률은 백테스트보다 20~40% 낮게** 나오는 게 일반적입니다.

---

**Q. 이 프로그램이 자동으로 주식을 사주나요?**

아닙니다. **알림만** 보내줍니다.
실제 매매는 직접 증권사 앱에서 해야 합니다.

---

**Q. Walk-forward에서 "거래부족"이 자꾸 나와요.**

검증 기간(뒤 4개월) 내 거래 횟수가 4회 미만이면 "거래부족"으로 표시됩니다.
이건 전략 버그가 아니라 해당 종목이 신호가 자주 나지 않는 것입니다.
거래 횟수가 너무 적으면 Sharpe 계산 자체가 의미 없기 때문에 정상 처리입니다.
