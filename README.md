## 2. 작동 원리

### 전체 흐름

```
[1단계] 데이터 수집
        yfinance + 업비트 API로
        103개 종목 주가 1년치를 한꺼번에 다운로드
        ※ yfinance 세션 워밍업(crumb 사전 획득)으로 401 오류 방지

        ↓

[2단계] 시장 국면 감지 (market_phase.py)
        KOSPI의 MA20·MA60·RSI·VIX를 보고
        지금이 상승장인지 하락장인지 판단
        → 국면에 따라 최소 점수(1~5점) 자동 설정
        ※ 선행지표 포함: 10년물 금리(^TNX), 공포지수(^VVIX→^VIX3M→VIX모멘텀)
        ※ VIX는 yfinance ^VIX 차단 시 FRED(미 연준) API로 폴백

        ↓

[3단계] 신호 계산 (signals.py)
        103개 종목을 병렬로 분석
        ① RSI < 40 미충족이면 즉시 차단 (나머지 지표 계산 안 함)
        ② 기술지표 5개(c2~c6) + 보너스 6개로 점수 계산
           ※ RSI(c1)는 게이트 전용 — 점수에 미포함
        ③ 독립 팩터 축 2개 이상 미충족이면 신호 차단 (5축 중)
           (RSI+볼린저만 = 같은 평균회귀축 1개 = 약신호 차단)
           5축: 추세/모멘텀/평균회귀/수급/상대강도(시장 대비)

        ↓

[4단계] 백테스트 검증 (signals.py)
        매수 신호 종목만 1년 과거 데이터로 검증
        아래 중 하나라도 해당되면 신호 차단:
        - 1년 백테스트 Sharpe < 0 (손실 전략)
        - Walk-forward 신뢰도 "낮음" (검증 구간 손실 또는 과적합)

        ↓

[5단계] 뉴스 수집 (data_loader.py)
        Gemini 실시간 검색 → Alpha Vantage → GNews → NewsAPI → yfinance
        5중 폴백 구조 / 영어 뉴스 자동 한국어 번역

        ↓

[6단계] AI 분석 (ai_analyst.py)
        Groq(llama-3.3-70b)가 1순위로 시장 이슈 + 신호 이유 생성
        Groq 실패 시 Gemini로 자동 폴백
        ※ 시장 브리핑(실시간 검색)만 Gemini 직접 사용 (Google Search 필요)

        ↓

[7단계] 리포트 조립 (scheduler_job.py)
        우선순위 점수(0~100점)로 종목 정렬 → TOP 3 선정
        진입 추천가 표시 (볼린저 하단 vs 현재가-ATR 중 높은 값)
        갭업 5% 이상이면 자동 제외
        상관계수 0.7 이상 종목 중 우선순위 낮은 쪽 차단 (섹터 집중 방지)

        ↓

[8단계] 텔레그램 전송 (telegram_bot.py)
        4000자씩 줄바꿈 경계에서 분할 전송 (마크다운 태그 경계 보호)
        신호 종목을 portfolio_signals.txt에 자동 기록
```

### 매수 신호 계산 방법

#### 전제 조건 (이것부터 통과해야 함)

| 조건 | 기준 | 이유 |
|------|------|------|
| **RSI < 40** | 미충족 시 즉시 차단 | RSI 40 이상은 과매도가 아닌 단순 하락 중. 과열 구간 매수 신호 방지 |

> RSI 40 이상인 종목은 나머지 조건이 모두 충족돼도 신호가 뜨지 않습니다.  
> RSI는 점수에도 포함되지 않습니다 (아래 참조).

#### 기본 조건 (각 1점, 최대 **5점**)

> RSI(c1)는 게이트 전용이라 기본 점수에 **포함되지 않습니다**.  
> RSI와 볼린저(c4)는 모두 "가격이 빠졌다"는 같은 정보라서 중복 카운트를 막기 위해 RSI는 게이트로만 씁니다.

| 번호 | 조건 | 쉬운 설명 |
|------|------|---------|
| c2 | 20일 이동평균 상승 중 | 한 달 평균 가격이 오르는 추세 |
| c3 | 오늘 거래량 > 평균 × 기준배수 | 평소보다 많이 거래 = 관심 급증 (VIX별 동적 조정) |
| c4 | 현재가가 볼린저 중심선 아래 | 통계적으로 평균보다 싼 가격 |
| c5 | MACD 히스토그램 음→양 전환 | 하락 에너지가 상승 에너지로 전환 |
| c6 | 변동성 돌파 + ADX ≥ 30 | 강한 추세와 함께 가격이 기준선 돌파 |

#### 보너스 조건 (각 +1점, 최대 +6점)

| 번호 | 조건 | 쉬운 설명 |
|------|------|---------|
| b0 | 독립 팩터 축 3개 이상 충족 | 서로 다른 근거가 3가지 = 허위 신호 감소 |
| b1 | RSI 다이버전스 | 가격은 내리는데 RSI는 올라 = 반등 예고 |
| b2 | MACD + 거래량 복합 강세 | 두 강한 신호가 동시에 발생 |
| b3 | 52주 신고가 95% 이상 | 1년 최고점 근처 = 저항선 없는 구간 |
| b4 | MACD + 스토캐스틱 동시 골든크로스 | 두 지표 동시 전환 = 허위 신호 감소 |
| b5 | CCI < -100 | 통계적으로 극단적 저평가 (RSI와 이중 확인) |
| b6 | 상대강도 +10%p 이상 | 시장을 크게 이기는 주도주 (오닐 CANSLIM) |

#### 독립 팩터 축 (매수 신호 필수 조건)

6개 조건 + 상대강도를 5개 독립 축으로 묶어 중복 카운트를 방지합니다.

| 독립 축 | 해당 조건 | 측정 대상 |
|--------|---------|---------|
| 추세축 | MA20(c2) OR 변동성돌파(c6) | 가격 방향 |
| 모멘텀축 | MACD(c5) | 추세 전환 속도 |
| 평균회귀축 | RSI(c1, 항상 충족) OR 볼린저(c4) | 저평가 수준 |
| 수급축 | 거래량(c3) | 참여자 관심 |
| **상대강도축** | **60일 수익률이 시장(KOSPI/S&P500) 초과** | **시장 대비 종목 고유의 힘** |

> **독립 축 2개 이상 미충족 시 매수신호 차단.**  
> RSI+볼린저만 충족한 종목은 둘 다 평균회귀축으로 실질 근거가 1개 → 차단.  
> 독립 축 3개 이상 충족 시 보너스(b0) +1점 추가.

#### 패널티

| 조건 | 효과 |
|------|------|
| 주간봉 MACD 음수 | 총점 -1점 (큰 흐름이 하락 추세) |
| 갭업 5% 이상 | 매수 목록에서 완전 제외 |
| 갭업 2~5% | 우선순위 점수 패널티 |
| 실적 발표 D-5 이내 | 우선순위 -5점 + 경고 표시 |

#### 백테스트 필터 (신호 차단 조건)

| 조건 | 차단 이유 |
|------|---------|
| 1년 Sharpe < 0 | 리스크 대비 수익이 마이너스인 전략 |
| WF 신뢰도 낮음 | 검증 구간 Sharpe < 0 또는 과적합 (Sharpe 차이 > 0.5) |

#### 시장 국면별 최소 점수

실행할 때마다 자동으로 결정됩니다.

| 국면 | 최소 점수 | 상황 |
|------|---------|------|
| 🚀 강한 상승 | **1점** | KOSPI가 강하게 오르는 중 |
| 📈 완만한 상승 | **2점** | 보통 상승 추세 |
| ↔️ 횡보 박스권 | **3점** | 오르지도 내리지도 않음 |
| 📉 조정 하락 | **4점** | 조금씩 내리는 중 |
| 🔴 급락 패닉 | **5점** | 급격히 하락 (거의 신호 없음) |

> RSI가 기본점수에서 제외(최대 5점)됨에 따라 임계값도 기존(2/3/4/5/6)에서 1씩 낮춰졌습니다.  
> 리포트에 표시되는 임계값과 실제 판정 임계값이 일치합니다.

#### 시장 국면 선행지표

후행 지표(MA, VIX)만 쓰면 하락장이 이미 시작된 다음에 경고가 옵니다.  
아래 선행지표 2개를 추가해 1~4주 일찍 경고합니다.

| 지표 | 경보 조건 | 의미 |
|------|---------|------|
| 10년물 국채금리(^TNX) | 한 달 새 +0.3%p 이상 급등 | 기업 대출비용 상승 → 주가 압박 선행 |
| 공포 지수(^VVIX→^VIX3M→VIX모멘텀) | ^VVIX 95↑ 또는 5일 +15% | 변동성 가속 = 선행 공포 신호 (구 ^PCCE/^PCALL은 yfinance 미존재로 제거) |

#### 우선순위 종합점수 (0~100점)

신호가 여러 개일 때 어떤 종목을 먼저 보여줄지 정하는 점수입니다.

| 항목 | 최대 점수 | 설명 |
|------|---------|------|
| 신호 점수 | 25점 | 기술지표 점수를 0~25로 환산 |
| RSI 점수 | 20점 | RSI 낮을수록 높은 점수 |
| Sharpe 점수 | 25점 | 백테스트 위험 대비 수익률 |
| 승률 점수 | 20점 | 백테스트 승률 |
| ADX 점수 | 10점 | 추세 강도 |

#### 진입 추천가

신호 발생 = 지금 당장 시가 매수가 아닙니다.  
알림에 지정가 대기 기준을 함께 표시합니다.

```
• 📌 진입 추천가: ₩285,000 (현재가 대비 -2.3%, 지정가 대기)
     └ 볼린저하단 ₩283,000 | 현재가-ATR ₩285,000
```

- **볼린저 하단**: 통계적 저점 지지선
- **현재가 - ATR**: 하루 변동폭만큼 눌렸을 때
- 두 값 중 현재가에 더 가까운 값을 추천 진입가로 제시합니다.

## 3. 파일 구조

```
main/
│
├── 📊 핵심 로직
│   ├── signals.py           매수/매도 신호, 백테스트, Walk-forward
│   │                        + 공용 헬퍼(거래량 임계값·52주 신고가·stoch/cci)
│   ├── indicators.py        RSI·MACD·볼린저·ATR·ADX·스토캐스틱·CCI·상대강도(RS)
│   ├── market_phase.py      시장 국면 5단계 + 선행지표(금리·공포지수)
│   └── scheduler_job.py     전체 실행 흐름, 리포트 조립, 벤치마크 주입
│
├── 📡 데이터·AI
│   ├── data_loader.py       주가 데이터 + 뉴스 수집 (5중 폴백)
│   ├── ai_analyst.py        Groq(1순위) + Gemini(폴백) + 번역
│   └── fundamental.py       기본적 분석 + 어닝 서프라이즈 + 기관/외국인 수급(KR)
│
├── 💼 포트폴리오
│   ├── portfolio.py         손익 알림, 트레일링 스탑, 주간 성과 리포트
│   ├── portfolio.txt        내가 보유한 종목 (직접 수정)
│   └── portfolio_signals.txt  신호 자동 기록 파일
│
├── 🌐 HTML 대시보드
│   ├── report_generator.py  텔레그램 리포트 → HTML 변환
│   └── docs/                GitHub Pages 배포 폴더
│       ├── index.html        리포트 목록 (최신순, 자동 생성)
│       └── YYYY-MM-DD-HHMM.html  실행별 리포트 (히스토리 누적)
│
├── ⚙️ 설정
│   ├── settings.txt         주요 파라미터 (RSI 기준, 손절 등)
│   ├── stocks.txt           감시할 종목 목록
│   ├── config.py            설정 파일 읽기
│   └── stocks_loader.py     종목·포트폴리오 파일 읽기
│
├── 📨 전송
│   ├── telegram_bot.py      텔레그램 메시지 분할 전송 (마크다운 태그 보호)
│   └── main.py              프로그램 시작점 (KST 실행 시각 기준 모드 분기)
│
├── 🔑 진단
│   └── check_keys.py        API 키 등록·작동 점검 (값 노출 없이 마스킹)
│
└── 🤖 자동화
    └── .github/workflows/
        ├── run_analysis.yml  메인 스케줄 (하루 3회 자동 실행)
        └── check_keys.yml    키 점검 전용 (수동 버튼, 코드 수정 불필요)
```

**방법 (코드 수정 불필요):**
1. GitHub 레포 → **Actions 탭**

3. 오른쪽 **"Run workflow"** 버튼 클릭

**출력 예시:**


**상태 의미:**
| 표시 | 의미 |
|------|------|
| ✅ | 키 정상 작동 |

> 키 값은 앞 4자리만 마스킹(`gsk_****`)되어 표시되므로 로그 노출 걱정이 없습니다.

## 5. HTML 대시보드 설정

텔레그램 메시지는 텍스트 서식만 지원해서 한눈에 보기 불편합니다. HTML 대시보드를 설정하면 실행할 때마다 **보기 좋은 웹 리포트**가 GitHub Pages에 자동 배포되고, 텔레그램에는 링크가 함께 전송됩니다.

```
텔레그램 메시지 하단에 추가되는 내용:
─────────────────────────
📊 HTML 대시보드
  2026-06-05-0944.html  ← 클릭하면 오늘 리포트 (예쁜 화면)
  전체 목록 보기        ← 클릭하면 날짜별 히스토리 목록
```

### 설정 방법 (최초 1회)

#### 1단계: GitHub Pages 활성화

레포지토리 → **Settings → Pages**  
→ Source: `Deploy from a branch`  
→ Branch: `main` / `docs`  
→ **Save**

잠시 후 `https://[계정명].github.io/[레포명]` 주소가 활성화됩니다.

#### 2단계: GH_PAGES_URL Secret 등록

Settings → Secrets and variables → Actions → **New repository secret**

| 이름 | 값 |
|------|---|
| `GH_PAGES_URL` | `https://[계정명].github.io/[레포명]` |

이 Secret을 등록하면 텔레그램 메시지에 HTML 링크가 자동으로 붙습니다.  
등록하지 않으면 HTML 파일은 생성되지만 링크는 전송되지 않습니다.

### 히스토리 관리

실행할 때마다 `docs/YYYY-MM-DD-HHMM.html` 파일이 쌓입니다.  
인덱스 페이지(`index.html`)에는 최근 90개(약 1달치)가 표시됩니다.

**특정 날짜 삭제**

```bash
git rm docs/2026-06-01-0900.html
git commit -m "리포트 삭제"
git push
```

**전체 초기화**

```bash
git rm docs/*.html
git commit -m "히스토리 초기화"
git push
```

> 초기화 후 다음 실행 때 `index.html`이 자동 재생성됩니다.

### 용량 안내

HTML 파일 1개 ≈ 50~100KB. 하루 3번 × 평일 250일 = 연간 약 75MB.  
GitHub 무료 플랜 용량 한도(1GB)에서 13년치를 저장해도 여유 있습니다.

## 6. 설정 변경 방법

`settings.txt` 파일을 메모장으로 열어서 숫자만 바꾸면 됩니다.  
바꾼 뒤 GitHub에 push하면 다음 실행부터 적용됩니다.

```
RSI_BUY=40
```
RSI가 이 값 **미만**인 종목만 매수 신호를 냅니다.  
미충족 시 신호가 완전 차단됩니다 (점수에 포함되지 않음).  
신호가 너무 없으면 45로 올려보세요. 50 이상은 권장하지 않습니다.

```
RSI_SELL=75
```
RSI가 이 값을 초과하면 매도 신호입니다.

```
STOP_LOSS=-5
```
진입가 대비 이 % 이상 내리면 손절 알림입니다.

```
TRAILING_STOP=8
```
매수 후 최고점 대비 이 % 내리면 청산 알림입니다.

```
TARGET_1=12
TARGET_2=25
```
분할 매도 목표입니다. 1차 12%, 2차 25% 수익 시 알림.

```
ADX_MIN=30
```
변동성 돌파 조건(c6)이 인정되는 최소 ADX 추세 강도입니다.  
낮출수록 신호가 늘지만 허위 신호도 증가합니다. 25~35 권장.

```
MAX_HOLDING_DAYS=30
```
이 일수 이상 보유 중이면 청산 권고 알림이 옵니다.

```
MAX_SIGNAL_DISPLAY=10
```
알림에 상세 표시할 최대 종목 수입니다.

```
BUY_SCORE_THRESHOLD=4
```
매수 최소 점수 기본값입니다. 실제 실행 시 시장 국면이 자동으로 덮어씁니다.  
국면별 실제 임계값: 강한상승 1점 / 완만한상승 2점 / 횡보 3점 / 조정 4점 / 패닉 5점

```
CORRELATION_MAX=0.7
```
매수 후보 종목 간 상관계수 상한입니다. 이 값 이상이면 우선순위 낮은 종목을 제외합니다.  
반도체 동시 신호처럼 위장된 집중 투자를 방지합니다. 0.8로 올리면 더 허용적, 0.6으로 낮추면 더 엄격합니다.

```
MAX_POSITIONS=8
```
동시 매수 권장 최대 종목 수입니다 (주식 기준, 크립토 별도). 초과하면 우선순위 낮은 종목은 제외됩니다.

```
예시:
005930.KS,10,68000,KRW

UPBIT:BTC,0.01,95000000,KRW
```

### portfolio_signals.txt — 신호 자동 기록

매수 신호가 날 때마다 자동으로 여기에 기록됩니다.

```
형식: 날짜,티커,종목명,신호점수,진입예상가,통화,보유일수,상태
```

**상태 종류:**

| 상태 | 의미 | 변경 방법 |
|------|------|---------|
| `신호발생` | 신호는 났지만 아직 사지 않음 | 자동 기록 |
| `보유중` | 실제로 산 경우 | **직접 수정** |
| `익절` | 목표가 도달 후 판 경우 | 직접 수정 |
| `손절` | 손절가 도달 후 판 경우 | 직접 수정 |
| `기간초과청산` | 30일 지나서 판 경우 | 직접 수정 |

**사용 방법:**
1. 신호가 오면 `신호발생`으로 자동 기록됩니다
2. 실제로 샀다면 해당 줄의 상태를 `보유중`으로 직접 바꿉니다
3. 30일이 지나면 자동으로 "청산 권고" 알림이 옵니다
4. 매주 일요일에 지난 주 신호들의 실제 수익률 요약이 옵니다

## 9. 자주 묻는 질문

**Q. 매수 신호가 하나도 안 나와요.**

상승장에서 RSI 40 미만인 종목이 없으면 정상입니다.  
국면이 "완만한 상승"이면 임계값이 2점으로 낮아지지만, RSI<40이면서 서로 다른 독립 팩터 축 2개 이상을 동시에 충족하는 종목 자체가 적습니다.  
신호가 계속 0개라면 `settings.txt`에서 `RSI_BUY=45`로 올려보세요. 50 이상은 권장하지 않습니다.

---

**Q. Walk-forward에서 "신뢰도 낮음"으로 차단되는 종목이 많아요.**

정상입니다. 거래 횟수가 적거나 검증 구간 Sharpe가 음수인 경우 차단됩니다.  
실전에서도 손실 날 가능성이 높은 전략을 걸러내는 것입니다.

---

**Q. AI 분석(Groq/Gemini)이 안 나와요.**

Groq가 1순위입니다. `GROQ_API_KEY`를 GitHub Secrets에 등록했는지 확인하세요.  
[console.groq.com](https://console.groq.com)에서 이메일만 입력하면 무료 발급됩니다(분당 30회).  
Groq도 실패하면 Gemini로 자동 폴백합니다. 둘 다 없으면 AI 분석 없이 리포트가 전송됩니다.

---

**Q. "Gemini 429 오류"가 자꾸 납니다.**

v5.4부터 Groq가 1순위로 변경됐습니다. `GROQ_API_KEY`를 등록하면 Gemini를 거의 호출하지 않아 429가 발생하지 않습니다.  
시장 뉴스 브리핑(실시간 검색)은 Gemini Google Search가 필요하므로 `GEMINI_API_KEY`도 함께 등록하세요.

---

**Q. 알림이 예약 시각과 다른 시간에 옵니다.**

GitHub Actions cron은 정시 보장이 없어 30분~2시간 지연이 흔합니다.  
프로그램이 실제 실행 시각을 보고 모드를 자동 판단하므로 기능상 문제는 없습니다.  
허용 윈도우: morning 08:30~10:30 / afternoon 14:30~17:30 / night 21:00~04:00.  
이 범위를 벗어나면(매우 드문 지연) 전체 분석 모드로 실행됩니다.

---

**Q. 알림에 뉴스가 안 보여요.**

뉴스는 5중 폴백으로 작동합니다.

1. Gemini Grounding (변동률 ±2% 이상 종목만, `GEMINI_API_KEY` 필요)
2. Alpha Vantage (미국 주식 전용, `ALPHAVANTAGE_API_KEY` 필요)
3. GNews (한국·미국 전종목, `GNEWS_API_KEY` 필요 — [gnews.io](https://gnews.io) 무료)
4. NewsAPI (GNews 폴백, `NEWSAPI_KEY` 필요 — [newsapi.org](https://newsapi.org) 무료)
5. yfinance (최후 수단, 불안정)

한국 주식 뉴스는 GNews(`GNEWS_API_KEY`)가 없으면 yfinance에만 의존해 뉴스가 0개로 올 수 있습니다.

---

**Q. 수급 데이터가 항상 "조회 실패"로 표시돼요.**

KRX(`data.krx.co.kr`)와 네이버금융은 GitHub Actions IP(해외)를 차단하는 경우가 있습니다.  
v5.6부터 **pykrx**를 1순위로 추가해(pykrx → KRX직접 → 네이버 → FDR → yfinance 5중 폴백) 성공률을 높였지만,  
pykrx도 결국 KRX 서버를 호출하므로 해외 IP 차단은 완전히 못 피합니다.  
완전한 해결은 한국 IP에서 실행(self-hosted runner)하는 방법뿐입니다.  
수급은 "있으면 보너스, 없으면 패스"로 설계돼 있어 조회 실패 시 다른 신호에 영향을 주지 않습니다.

---

**Q. 로그에 "^VIX: possibly delisted"가 떠요.**

VIX 심볼이 폐지된 게 아니라, yfinance의 지수 다운로드 엔드포인트가 GitHub Actions 해외 IP에서 차단된 것입니다.  
v5.10부터 ^VIX가 막히면 **FRED(미국 연준) 공개 API**로 VIX를 가져옵니다. IP 차단이 없고 키도 불필요합니다.  
정상 동작 시 로그에 `💹 VIX: 18.9 (FRED)` 또는 `(yfinance)`로 출처가 표시됩니다.  
둘 다 실패하면 기본값 15.0으로 처리되며, 이 경우 시장 국면 판단이 다소 부정확해질 수 있습니다.

---

**Q. HTML 대시보드 링크가 텔레그램에 안 붙어요.**

`GH_PAGES_URL` Secret이 등록되지 않은 경우입니다.  
Settings → Secrets에서 `GH_PAGES_URL` 값을 `https://[계정명].github.io/[레포명]` 형식으로 등록하세요.

---

**Q. 알림이 너무 많아요.**

`settings.txt`에서 두 가지를 조정하세요.

- `MAX_SIGNAL_DISPLAY=5` (상세 표시 줄이기)
- `BUY_SCORE_THRESHOLD=5` (기준 높이기 — 국면별 자동 설정을 덮어씀)

---

**Q. 어닝 서프라이즈는 어떻게 점수에 반영되나요?**

회사가 발표한 실제 EPS가 전문가 예상(컨센서스)을 이겼는지 봅니다.

- 4분기 연속 예상 상회 → +1.5점 (꾸준한 실적주)
- 2분기 이상 연속 상회 → +1.0점
- 최근 분기 +5% 이상 깜짝 상회 → +0.5점
- 최근 분기 -10% 이상 하회 → -1.0점 (실적 악화 경고)

미국주식은 yfinance에서 잘 제공되지만, 한국주식은 EPS 추정치 데이터가 빈약해 대부분 빈값으로 나올 수 있습니다(없으면 점수 영향 없음). 가격 지표(RSI·MACD)나 밸류에이션(PER)과 겹치지 않는 독립 정보라, 같은 점수라도 실적이 뒷받침되는 종목을 우대합니다.

**Q. 백테스트 수익률이 실제와 다를 것 같아요.**

맞습니다. 백테스트는 신호 발생 당일 종가에 산다고 가정하는데  
실제로는 다음 날 시가에 삽니다.  
갭이 1~3% 발생하고 슬리피지도 더 크게 나옵니다.  
**실제 수익률은 백테스트보다 20~40% 낮게** 나오는 게 일반적입니다.  
반드시 페이퍼 트레이딩으로 먼저 검증하세요.

### v5.5 — RSI 게이트 전용화 및 전략 일관성

#### 변경 내용

| 항목 | 변경 전 | 변경 후 |
|------|-------|-------|
| RSI 역할 | 게이트 + 기본점수 1점 | 게이트 전용 (점수 미포함) |
| 기본점수 최대 | 6점 (c1~c6) | **5점 (c2~c6)** |
| 국면별 임계값 | 2/3/4/5/6점 | **1/2/3/4/5점** |
| 매수신호 조건 | 점수 ≥ 임계값 | **점수 ≥ 임계값 AND 독립축 ≥ 2** |
| 백테스트 VIX 거래량 | 항상 1.3배 고정 | **VIX별 동적 적용** (≥30: 2.0배, ≥20: 1.6배) |
| 백테스트 보너스 | b1~b3 3개 | **b0~b5 6개 (calc_signals와 완전 동일)** |

#### 이유

RSI(c1)와 볼린저(c4)는 둘 다 "가격이 빠졌다"는 평균회귀축 정보를 공유합니다.  
RSI를 게이트로 쓰면 c1은 통과 구간에서 항상 True → 점수에 넣으면 자동 +1점이 생겨 임계값이 왜곡됩니다.  
c1을 점수에서 제거해 "RSI+볼린저만으로는 신호 불가"를 명확히 하고, 임계값을 1씩 낮춰 동일한 선별력을 유지합니다.
